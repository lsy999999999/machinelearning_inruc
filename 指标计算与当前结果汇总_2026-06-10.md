## 1. F1 怎么算

devkit 先取预测类别：

```text
y_pred = argmax(probabilities)
```

然后按 9 个类别分别计算：

```text
precision_c = TP_c / (TP_c + FP_c)
recall_c    = TP_c / (TP_c + FN_c)
F1_c        = 2 * precision_c * recall_c / (precision_c + recall_c)
```

最终使用 Macro-F1：

```text
macro_f1 = mean(F1_0, F1_1, ..., F1_8)
```

当前：

```text
macro_f1 = 0.983716900
```

## 2. Faith 怎么算

devkit 先把 relevance map 变成非负并归一化：

```text
relevance = max(relevance, 0)
relevance = relevance / sum(relevance)
```

然后按 relevance 排序做 top-k 删除/插入，比例为：

```text
0%, 10%, 20%, ..., 100%
```

Deletion：从原始输入开始，逐步把 top-k 重要位置替换成 baseline，越快让预测置信度下降越好。

Insertion：从 baseline 开始，逐步插入 top-k 重要位置，越快恢复预测置信度越好。

最终：

```text
faith = (insertion_auc + (1 - deletion_auc)) / 2
```

当前：

```text
deletion_auc  = 0.122045794
insertion_auc = 0.713854611
faith         = 0.795904409
```

## 3. Simplicity 怎么算

devkit 根据 ONNX 模型复杂度计算：

```text
penalty = parameter_count / 1,000,000
        + operator_count / 1,000
        + onnx_size_mb / 50

simplicity = 1 / (1 + penalty)
```


## 6. 当前主要短板

从每类 F1 看，最低的是：

```text
CWF = 0.972973
ORF = 0.973199
RCF = 0.976703
IRF = 0.978240
```

主要问题不是整体分类能力，而是这些类别之间的边界：

```text
CWF -> ORF: 22 个
IRF -> ORF: 7 个
HEA -> RCF: 10 个
```

所以如果后面还想提高 F1，重点应放在组合故障、内外圈故障、齿根裂纹这几类的混淆，而不是重新训练一个更大的整体模型。
