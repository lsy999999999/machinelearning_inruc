我查了几类“物理 + 机器学习”的旋转机械/齿轮箱/轴承故障论文。结论是：确实还有优化空间，但不应该上真正复杂的 PINN/PDE 大
  模型；更适合把论文里的物理思想继续折叠成我们现在的离线 weights_9x8x100 候选，再用 full devkit coordinate 选。

  最有用的启发

  1. 特征频率约束 / 物理 loss
     有论文用 BPFO/BPFI 等轴承特征频率做 physics-informed loss，在变工况下提升鲁棒性，核心不是大模型，而是“预测和物理特
     征频率不一致就惩罚”。参考：Physics-Informed Multimodal Bearing Fault Classification under Variable Operating
     Conditions using Transfer Learning，里面强调特征频率和跨工况泛化。
     来源：https://arxiv.org/abs/2508.07536

     对我们启发：现在没有完整 speed/load metadata，但可以做 pseudo-order / pseudo-lag prior。也就是不用真实转速，直接从
     信号自相关、包络峰间距、torque 周期性估计“周期 lag”，给 ORF/IRF/BWF/CWF 做周期 comb relevance。

  2. Classifier-guided blind deconvolution
     ClassBD 这类方法把盲反卷积、kurtosis、l2/l4 norm 和 CE loss 结合，用分类器指导提取冲击特征。
     来源：https://arxiv.org/abs/2404.15341

     对我们启发：我们之前试过 kurt_abs、diff、combo_impulse，但还不够“classifier-guided”。下一步应该按 高置信正确样本 /
     边界错例 / class 6/7/8 分组重新生成 impulsiveness proxy，而不是全局一个 kurtosis prior。

  3. Cyclostationary / Spectral Correlation Density
     SCD 思路利用旋转机械信号的循环平稳性，比普通 FFT 更适合非平稳冲击。
     来源：https://arxiv.org/abs/2509.16580

     对我们启发：窗口只有 100 点，直接做完整 SCD 不现实，但可以做轻量版：
     autocorr lag energy + envelope periodicity + channel-family weighting。这可以变成新的离线权重候选，不加 ONNX 算子。

  4. 能量算子 / CEEO
     齿轮箱论文里有 calculus enhanced energy operator，用来在噪声里保留特征频率，再接传统分类器。
     来源：https://arxiv.org/abs/2308.14174

     对我们启发：可以做 Teager/energy-operator relevance prior，尤其对齿轮冲击类和组合故障类。我们现在的
     sqrt(abs(input)) 是幅值型，能量算子可能能更好抓“瞬时冲击”。

  我认为下一步最值得做的 3 个实验

  1. Periodic Comb / Pseudo-Order Prior
     针对 class 6/7/8，按自相关峰或 envelope peak spacing 生成周期 comb relevance row，然后和当前 smooth6_c678、
     upgrade、phys_all/top 做 row-coordinate。

  2. Classifier-Guided Impulse Proxy
     只用当前模型高置信正确样本和 CWF/IRF/ORF 边界样本，重新统计：
     diff energy / local peak / Teager energy / kurtosis / l2-l4 impulsiveness。
     重点不是新增模型，而是生成新的 weights_9x8x100 候选。

  3. Pseudo-Condition Robust TimeREISE
     既然现在 prepared metadata 没有 condition_id，就按信号统计伪分组：
     RMS level / torque mean / dominant lag / envelope periodicity。
     每组算一套 TimeREISE/marginal 权重，再用 median/trimmed aggregate。这个比普通 trimmed 更贴近“跨工况稳健”。

  不建议

  - 不建议现在上复杂 CNN/Transformer/PINN 主模型。当前分类已经很高，训练分支很可能损 faith 和 simplicity。
  - 不建议把 SCD/盲反卷积直接放进 ONNX。应该离线生成权重候选，保持提交图简单。
  - 不建议继续只调全局 gamma。现在收益已经主要在 class row 和物理 proxy 的细粒度组合。

  Teager energy、l2/l4 impulsiveness、边界样本重加权，全都折叠成候选 ONNX，然后接 full row-coordinate。
