# GearXAI 冲分与物理先验调研报告

日期：2026-06-10

## 1. 评分口径变化

老师确认当前分数按三项加权：

```text
score = 0.6 * macro_f1 + 0.3 * faith + 0.1 * simplicity
```

这和官方 devkit 里原来的 `0.4 * faith + 0.4 * mechanical + 0.2 * simplicity` 口径不同。本报告下面的冲分建议优先服从老师给出的 `f1/faith/simplicity` 三项权重；物理先验主要作为提高 faith、降低 hidden split 风险、增加汇报创新性的手段。

当前本地最佳候选仍建议保持为：

```text
runs/final_candidates/logic_timereise_class_candidate_selection_bestproxy_submission.zip
```

对应本地 5k devkit validation：

| candidate | macro_f1 | faith | deletion_auc | insertion_auc | simplicity | 新加权分 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| classsel_pred | 0.983717 | 0.788796 | 0.130383 | 0.707975 | 0.901619 | 0.917031 |

分数换算：

| 变化 | 对总分影响 |
| --- | ---: |
| `macro_f1 +0.005` | `+0.0030` |
| `macro_f1 -0.005` | `-0.0030` |
| `faith +0.010` | `+0.0030` |
| `simplicity -0.020` | `-0.0020` |

结论很直接：

1. `macro_f1` 权重最高，但当前已经接近饱和，不能为了新解释模块牺牲分类器。
2. 现在最合算的是“分类输出不变，只改 relevance map”，因为这样 `macro_f1` 稳定、`simplicity` 基本不变。
3. 任何能让 `faith +0.005` 的 relevance-only 方法，都会带来约 `+0.0015` 总分；这个幅度已经值得做。
4. 在线加入复杂物理算子如果让 simplicity 掉 `0.02`，需要 faith 涨至少 `0.0067` 才刚好抵消。

## 2. 当前方案为什么已经走对方向

本地 devkit 的 faithfulness 计算逻辑是：

```text
faith = (insertion_auc + (1 - deletion_auc)) / 2
```

评估过程会把 relevance map 归一化，然后按 top-k 位置做删除和插入，比例是：

```text
0%, 10%, 20%, ..., 100%
```

所以 relevance 的绝对值不重要，重要的是“排序是否把真正影响预测置信度的位置排在前面”。

当前 TimeREISE 路线已经抓住了这一点：

```text
relevance = sqrt(abs(input)) * soft_class_weighted_factor
```

其中 `factor[class, channel, time_bin]` 离线统计并折叠进 ONNX。这样有三个优点：

1. 分类概率不变，`macro_f1=0.983717` 保住。
2. ONNX 结构简单，`simplicity≈0.9016` 保住。
3. 解释图只通过离线权重改变排序，正好对齐 deletion / insertion faith。

已经验证过的有效改进包括：

| 路线 | faith | deletion_auc | insertion_auc | 结论 |
| --- | ---: | ---: | ---: | --- |
| robust/offline contrast 早期版本 | 0.752061 | 0.186942 | 0.691064 | 有效但不是最终 |
| marginal b10 | 0.770607 | 0.146884 | 0.688098 | 明显改善 deletion |
| power ensemble | 0.783607 | 0.140239 | 0.707453 | insertion 也被拉高 |
| class-specific candidate selection | 0.788796 | 0.130383 | 0.707975 | 当前最佳 |

## 3. 论文调研后的可借鉴方法

### 3.1 RISE / TimeREISE：扰动归因是当前路线的理论基础

RISE 用随机 mask 探测黑盒模型输出变化，并用 deletion / insertion 评价解释质量；TimeREISE 把类似思想迁移到时间序列。我们的做法本质上是把 TimeREISE 改成了更适合 GearXAI 的低复杂度版本：

```text
随机/边际扰动统计 -> class/channel/time_bin factor -> 折叠为简单 ONNX relevance
```

可继续借鉴的点：

1. 多尺度 mask：不要只固定 b10，可以额外做 b5/b10/b20 的候选，再用逐类 selection 选择。
2. 随机 coalition：不只删除单块，而是随机保留/删除多个块，估计更接近 Shapley/Owen value 的边际贡献。
3. 评估粒度对齐：官方 top-k 是 10 个比例点，所以 b10 是主尺度，其他尺度只作为补充候选。

### 3.2 WindowSHAP / GroupSegment-SHAP：把通道-时间块当作解释单位

WindowSHAP 说明时间序列解释不应该把每个时间点完全独立看待，而应把连续窗口作为 attribution 单位。2026 年的 GroupSegment-SHAP 进一步把“特征组 + 时间片段”作为玩家，适合多变量时间序列。

这和 GearXAI 的输入非常匹配：

```text
8 channels x 100 time steps
=> 4 个物理通道组 x 10 个时间块
=> group-segment players
```

建议实现一个轻量版 `GroupSegment-TimeREISE`：

1. 通道组：
   - motor vibration
   - RGB gearbox vibration x/y/z
   - torque
   - PGB gearbox vibration x/y/z
2. 时间组：
   - 10 个长度为 10 的时间块
3. 随机采样若干 group-segment coalition。
4. 记录预测类别置信度变化。
5. 聚合为 `factor[class, channel, time_bin]`。
6. 最终仍导出同样简单的 ONNX。

这个方向比“逐 cell SHAP”现实得多，因为 800 个 cell 太多，40 个 group-segment player 还可以做近似采样。

### 3.3 DynaMask / Learning Perturbations：可做离线 teacher，不适合直接放进最终模型

DynaMask 和后续 Learning Perturbations 都是学习一个 mask，让解释图在删除/保留任务中更忠实。这类方法理论上很适合 faithfulness，但直接用于最终提交有两个问题：

1. 每个样本都要优化 mask，推理太慢，不适合 ONNX 提交。
2. 如果把 mask network 加到模型里，operator count 和参数量上升，simplicity 会下降。

比较稳的用法是：

```text
学习型 mask 只在离线阶段生成 teacher relevance
teacher relevance -> 蒸馏成 class/channel/time_bin factor
最终 ONNX 不包含 mask network
```

也就是说，它适合作为后续“离线生成候选解释图”的方法，不适合直接作为最终模型结构。

### 3.4 ID-ExpO：说明可以直接优化 insertion/deletion，但当前不建议重训分类器

ID-ExpO 用可微 insertion/deletion 正则提高解释 faithfulness。它说明“按评价指标训练解释”是合理方向，但我们现在不建议重新训练主分类器。

原因：

```text
teacher score 里 macro_f1 权重 0.6
当前 macro_f1 已经 0.983717
任何分类器回退都会很亏
```

如果要借鉴 ID-ExpO，只建议训练一个小的离线 relevance teacher，最后仍蒸馏到静态 factor。

### 3.5 ShapeX / shapelet 思想：故障冲击片段可以当作可解释 motif

ShapeX 的核心思想是：时间序列分类往往由关键 subsequence/shapelet 驱动，而不是孤立时间点驱动。齿轮箱振动里，缺齿、裂纹、磨损、轴承内外圈故障往往会产生局部冲击、调制或周期性片段，所以 shapelet 思想有物理直觉。

可迁移做法：

1. 对每类故障从高置信样本里找典型高能/高峭度片段。
2. 用这些片段作为 class-specific motif。
3. 计算每个样本各时间块和 motif 的相似度。
4. 不在线输出复杂相似度网络，而是把 motif 命中的通道-时间统计折叠为离线 factor 候选。

这条线更适合汇报“创新性”，但短期冲分优先级低于 group-segment 扰动和物理先验融合。

## 4. 实际物理是否有用

我的判断：有用，但要小心用。

GearXAI 是物理故障诊断，真实故障信号不会随机出现。齿轮/轴承类振动通常和以下物理现象有关：

| 物理现象 | 对 relevance 的启发 |
| --- | --- |
| 局部冲击 | 高峰值、高峭度、相邻差分能量大的位置更可能重要 |
| 齿轮啮合/边带调制 | 解释图不应只集中在单点，而应呈现某种时间周期或相邻块结构 |
| 轴承内圈/外圈/滚珠故障 | envelope energy、重复冲击、调制频率附近能量可能有用 |
| 变速/变载工况 | 直接硬编码频率风险大，应按 RMS、torque、主频等伪工况做 robust 聚合 |
| 多传感器布局 | RGB/PGB 三轴振动、motor vibration、torque 的重要性不应完全同质 |

但是这里有一个关键限制：公开数据窗口只有 100 点，采样率本地 devkit 写的是 5120 Hz，一个窗口约 19.5 ms；同时我们没有明确的转速、齿数、轴承几何参数。因此不建议硬编码具体 GMF、BPFO、BPFI 数值。

更稳的物理路线是“数据驱动的物理 proxy”：

```text
不用硬编码具体故障频率
只用物理上合理的统计量
再通过 devkit faith 检验是否真有用
```

## 5. 最推荐的物理先验实现

### 5.1 离线物理 prior，不增加 ONNX 复杂度

为每个样本、通道、b10 时间块计算：

| proxy | 计算方式 | 物理含义 |
| --- | --- | --- |
| absolute energy | `mean(abs(x))` 或 `mean(x^2)` | 振动强度 |
| diff energy | `mean((x[t]-x[t-1])^2)` | 冲击/突变 |
| peakness | `max(abs(x)) / rms(x)` | 尖峰程度 |
| kurtosis proxy | 四阶矩 / 二阶矩平方 | 冲击性 |
| local envelope | `sqrt(smooth(x^2))` 或 abs + smoothing | 包络强度 |
| periodic proxy | 短 lag autocorr 的最大值 | 重复冲击/调制 |
| torque coupling | torque 块能量与振动块能量相关性 | 负载相关性 |

然后按预测类别或真实类别聚合：

```text
phys_prior[class, channel, time_bin]
```

最终融合：

```text
R_final = normalize((R_current + eps)^a * (phys_prior + eps)^gamma)
```

建议小网格：

```text
a = 0.98, 1.00, 1.02
gamma = 0.001, 0.002, 0.005, 0.010, 0.020
```

为什么 gamma 要小：

1. 当前 `classsel_pred` 已经很强。
2. 之前机械先验 `gamma=0.005` 只到 `faith=0.788486`，略低于当前最佳。
3. 物理 prior 的价值可能更多体现在 hidden robustness 和汇报解释，不一定直接拉高 public faith。

### 5.2 按故障类别使用不同物理 prior

类别名称来自本地数据解析：

| label | fault name |
| --- | --- |
| HEA | healthy |
| CTF | chipped tooth fault |
| MTF | missing tooth fault |
| RCF | root crack fault |
| SWF | surface wear fault |
| BWF | ball fault |
| CWF | combination fault |
| IRF | inner race fault |
| ORF | outer race fault |

建议的 class-specific prior：

| 类别 | prior 侧重 |
| --- | --- |
| HEA | 不要过度尖锐，保持平滑/低冲击；避免把正常局部高幅值误当故障 |
| CTF/MTF/RCF/SWF | 齿轮相关三轴振动、局部冲击、周期性和边带 proxy |
| BWF/IRF/ORF | envelope、peakness、kurtosis、重复冲击 proxy |
| CWF | 允许多个通道组共同高亮，避免过度稀疏 |

这可以和当前逐类 candidate selection 合并：

```text
每个 class 不只选 marginal / power / robust
还可以选 phys_g001 / phys_g002 / phys_g005 等候选
```

这种做法几乎不影响 simplicity，因为最终仍然只是 `weights_9x8x100`。

### 5.3 伪工况 robust 聚合

Gearbox 数据可能有速度/负载变化，论文和数据集调研都强调 variable operating conditions 是真实难点。公开窗口未必给出完整工况字段，但可以用 proxy 分组：

```text
condition_key = quantile_bin(
    total_rms,
    torque_rms,
    dominant_frequency_proxy,
    motor_to_gearbox_energy_ratio
)
```

实现方式：

1. 按伪工况分组统计 TimeREISE / marginal factor。
2. 对每组 factor 做 mean / median / trimmed mean。
3. 生成 robust factor 候选。
4. 再进入 class-specific candidate selection。

这条线的本地收益可能不如直接 marginal，但对 hidden split 和汇报合理性更强。

## 6. 短期实验优先级

### 第一优先级：物理 prior 小权重融合

预计成本低，最适合马上做。

```text
输入：当前 classsel_pred weights
新增：phys_prior[class, channel, time_bin]
搜索：gamma = 0.001 / 0.002 / 0.005 / 0.010 / 0.020
保留条件：faith 不低于 0.7888，或加权分超过 0.917031
```

如果 faith 基本持平但解释图更符合故障物理，也值得保留为汇报候选。

### 第二优先级：GroupSegment-TimeREISE

预计成本中等，创新性高。

```text
players = channel_group x time_bin
channel_group = motor / RGB / torque / PGB
time_bin = 10 blocks
method = random coalition perturbation
output = class/channel/time_bin factor
```

保留条件：

```text
faith >= 0.790
或者与当前最佳持平但 deletion/insertion 曲线更均衡
```

### 第三优先级：按新总分重做 candidate selection

当前逐类 selection 主要围绕 faith / proxy。现在老师权重已经明确，可以重算一个简单目标：

```text
score_class = 0.3 * local_faith_class
```

因为分类器不变、simplicity 基本不变，逐类选择本质上还是选 local faith 最高。但如果某个候选让整体 ONNX operator/size 变化，则要加入 `0.1 * simplicity`。

建议候选池：

```text
current classsel rows
power ensemble rows
marginal b10 rows
robust rows
phys_g001 / phys_g002 / phys_g005 rows
groupsegment rows
```

### 第四优先级：学习型 mask teacher

只建议有额外时间再做。

```text
DynaMask / Learning Perturbations style mask
offline optimize on validation/train subset
distill to class/channel/time_bin factor
```

不建议把 mask network 放进最终 ONNX。

## 7. 不建议优先做的方向

| 方向 | 不优先原因 |
| --- | --- |
| 重新训练大分类器 | `macro_f1=0.983717` 已很高，回退风险大 |
| Transformer / 更深 CNN | 可能涨不了 F1，simplicity 明显吃亏 |
| 在线 FFT/STFT/Hilbert/TopK 解释模块 | 物理上好看，但 ONNX operator_count 增加，simplicity 可能掉 |
| 硬编码 BPFO/BPFI/GMF | 没有轴承几何、齿数、转速，硬编码容易错 |
| 只优化 deletion | 可能压低 deletion_auc，但 insertion_auc 会崩，faith 不一定涨 |
| 只看 public validation | 容易过拟合 relevance 排序，hidden 不稳 |

## 8. 汇报里可以强调的创新点

1. 从“训练一个更大模型”转向“稳定分类器 + 解释图优化”，这是由评分权重和 devkit 行为共同决定的。
2. 提出了 TimeREISE 的 GearXAI 低复杂度变体：离线扰动统计，在线只保留简单 relevance 公式。
3. 使用 official faithfulness 的 top-k deletion/insertion 机制反向设计 b10 marginal gain。
4. 通过 power ensemble 和 class-specific candidate selection，在不改变分类、不增加复杂度的情况下提高 faith。
5. 物理层面不是简单加 FFT，而是把冲击、包络、峭度、周期性、传感器布局等机械知识折叠成离线 prior，兼顾 faith 和 simplicity。
6. 后续的 GroupSegment-TimeREISE 可以解释为“机械通道组 + 时间片段”的 Shapley/扰动近似，比单点 saliency 更符合振动诊断。

## 9. 最终建议

当前最稳策略：

```text
主提交：logic_timereise_class_candidate_selection_bestproxy_submission.zip
冲分实验 1：phys_prior 小 gamma 融合
冲分实验 2：GroupSegment-TimeREISE
冲分实验 3：把新候选加入 class-specific selection
```

物理想法值得做，但不要做成复杂在线模型。最推荐的表达是：

```text
我们把物理知识用作离线 relevance 先验，而不是额外分类网络。
这样既利用了齿轮箱故障的实际机理，又不牺牲 F1 和 simplicity。
```

如果只剩很少时间，优先保当前最佳提交，并把物理 prior 作为“已设计、部分验证或下一步工作”写进报告；如果还有 1-2 天，先实现 `phys_prior_g001/g002/g005`，只要 faith 不掉，就可以作为更有解释性的备用候选。
