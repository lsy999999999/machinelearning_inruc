# GearXAI 来源与文献优化思路

日期：2026-06-13

## 1. 先看当前结构

当前仓库已经走到比较合理的位置：

- 主分类器用官方 `LogicLSTM`，本地 `macro_f1≈0.9837`，已经很高。
- 主要优化都集中在 `relevance_map`，通过 `TimeREISE`、marginal gain、物理 proxy、class-specific selection、row-coordinate search 提升 faith。
- 最强路线是不改分类概率、不增加 ONNX 复杂度，只折叠 `weights_9x8x100`。
- 当前最佳包：`runs/final_candidates/logic_timereise_row_coordinate_ext_bestproxy_submission.zip`，本地 `faith=0.795904`，`simplicity=0.901619`。

所以后续不建议先重训大模型。最值钱的方向仍是：保持 F1 和 simplicity，继续改 relevance 排序。

## 2. 比赛来源判断

GearXAI 看起来不是单纯由一篇论文直接引出的比赛，而是：

- 赛题：`GearXAI: An Explainable Neuro-Symbolic Gearbox Fault Diagnosis Challenge`
- 数据：`edi45/gearxai-dds-seu`，基于 SEU / DDS 传动系统齿轮箱数据。
- 官方 baseline：devkit 内置 `logic_lstm.onnx`、`logic_transformer.onnx` 等。
- 数据卡提到的直接支撑论文包括：
  - `LogicLSTM: Logically-driven long short-term memory model for fault diagnosis in gearboxes`
  - `Rule-Guided Transformers for Dynamic Knowledge Adaptation in Rotating Machinery Fault Diagnosis`

我没有找到 GearXAI 赛题本身稳定的公开 citation graph。尤其 RGT 是 2026 论文，引用它的文章大概率还很少。因此更实际的文献链是三条：

1. `LogicLSTM / Rule-guided`：神经符号、规则引导诊断。
2. `RISE -> TimeREISE -> Time-series SHAP / mask`：解释图和 deletion/insertion faith。
3. `spectral kurtosis / cyclostationarity / shapelet`：机械故障物理先验。

## 3. 看过的论文和可借鉴点

| --- | --- | --- | --- |
| 1 | LogicLSTM, JMS 2024 | 用逻辑规则引导 LSTM 做齿轮箱故障诊断 | 当前 baseline 来源；继续保分类器，不轻易重训 |

| 4 | TimeREISE, 2022 | 把 RISE 思路迁移到时间序列解释 | 已经是主线；可做多尺度和随机组合块 |
| 5 | DynaMask, 2021 | 学一个动态 mask 解释多变量时间序列 | 适合做离线 teacher，不适合直接塞进提交模型 |

| 9 | GroupSegment-SHAP, 2026 | 用“特征组 + 时间片段”作为 Shapley player | 最值得借鉴：8 通道可按传感器组，100 点按 10 块 |

## 4. 最值得做的优化方向

### A. GroupSegment-TimeREISE

把解释单位从单个 `channel x bin` 改为“通道组 x 时间块”：

- 通道组：motor、RGB 三轴、torque、PGB 三轴。
- 时间块：先用 10 块，必要时补 b5/b20。
- 随机采样多个 group-segment coalition。
- 统计 deletion / insertion gain。
- 最终仍折叠成 `weights_9x8x100`。

这个方向最贴合 GroupSegment-SHAP，又不增加 ONNX 复杂度。

### B. Multi-scale b5/b10/b20 row selection

官方 faith 是 `0%,10%,...,100%` top-k，b10 是主尺度。但不同故障可能靠短冲击或长片段：

- b5：更粗，可能保护 insertion。
- b10：当前主尺度。
- b20：更细，可能抓局部冲击。
- 不做全局平均，继续用 class row selection / coordinate search。

### C. Motif / shapelet prior

从高置信样本里找每类典型片段：

- 齿轮类：CTF / MTF / RCF / SWF 的局部冲击、重复峰。
- 轴承类：BWF / IRF / ORF 的 envelope、kurtosis、周期冲击。
- CWF：允许多通道多片段，不要过度稀疏。

短期不要在线算相似度。先把 motif 命中率统计成离线权重候选。

### D. Offline learned mask teacher

借 DynaMask / Learning Perturbations：

- 用小 mask 网络只在离线阶段训练。
- 目标直接对齐 deletion/insertion。
- 训练完把 teacher relevance 聚合成 `class, channel, bin` 权重。
- 提交模型不带 mask 网络。

风险是耗时较高，但理论上可能比手写 proxy 更强。

### E. Condition-aware robust aggregation

当前本地 `prepared_hf_val5k/validation_metadata.jsonl` 字段太少，不能直接做真实工况分组。但 HF 原始 `windows_100` 数据卡里有 `condition_id / speed_hz / load_nm / regime`，值得重新导出一次带工况字段的 validation metadata。

如果短期不重新导出，可以先用伪工况：

- RMS / crest factor / dominant lag / torque energy 分桶。
- 每桶统计一套权重。
- 最终用 mean / median / trimmed mean 聚合。

这条线不一定涨 public faith，但可能降低 hidden split 风险。

## 5. 优先级

1. 先做 `GroupSegment-TimeREISE`，保持 b10，直接复用现有 marginal/selection 框架。
2. 加 b5/b20 候选，只让 row-coordinate 决定是否采纳。
3. 做 shapelet/motif 离线 prior，作为候选权重加入 class selection。
4. 有时间再做 learned mask teacher；不要先改最终 ONNX 结构。
5. 不建议现在重训 Transformer/CNN 主模型，除非完整 devkit teacher score 明确超过当前 best。
